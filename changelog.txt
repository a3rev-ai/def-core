*** Digital Employee Framework - Core Changelog ***

= 1.2.7 - 2026/03/19 =
* Fix: Staff AI Share recipients dropdown now shows all users with Staff or Management access (was only showing site admin)
* Fix: Staff AI Share emails now include Reply-To header with the sharing user's email address
* Security: Share endpoint channel forced server-side — prevents confused-deputy escalation channel switching
* Security: Token picker DOM rendering switched from innerHTML to textContent/createElement (XSS prevention)
* Improved: Share recipients show display names — "Name (email)" format in chips and dropdown
* Improved: Current user excluded from own Share recipient list (self-exclusion)
* Tweak: Recipient picker data separated — allowed_recipients (policy) vs recipient_options (UI display)

= 1.2.6 - 2026/03/19 =
* Improved: Composer staging area redesigned — single image shows as 72px thumbnail tile, multiple attachments show compact 56px tiles + taller file chips
* Improved: Upload progress spinner — circular CSS spinner overlay on image tiles and file icons during upload, replaces flashing border animation
* Fix: Click image preview to open full size — now works correctly (converts data URL to blob URL for browser compatibility)
* Security: Click-to-open restricted to safe raster image types only (PNG, JPEG, GIF, WebP) — SVG excluded
* Tweak: File chips redesigned — taller layout (56px) with 28px icon, filename and size stacked vertically
* Tweak: Blob URLs revoked after use to prevent memory leak accumulation

= 1.2.5 - 2026/03/19 =
* Improved: Staff AI file uploads now start immediately on attach (not on Send) — eliminates 3-4 second hang time
* Improved: Uploaded images show as 384px inline previews in chat thread (click to open full size)
* Improved: Upload timeout handling — files stuck uploading after 30s marked as failed with user-facing error
* Security: Image thumbnail URLs validated with data:image/ prefix check before rendering
* Tweak: Non-image files retain existing file indicator chip (no regression)

= 1.2.4 - 2026/03/17 =
* Fix: OAuth PKCE storage — replaced set_transient() with update_option() for reliable PKCE verifier storage
* Security: Hardened PKCE expiry check — fail closed on missing or malformed expires_at

= 1.2.3 - 2026/03/16 =
* Diagnostic: WP-side OAuth instrumentation — SHA256 hash tracing for transient write, readback, and callback lookup
* Diagnostic: Transient storage verification — immediate readback test after set_transient()
* Diagnostic: Callback logging — transient lookup result, ext object cache status, blog_id, cookie user
* Note: All diagnostic work fully gated behind DEF_OAUTH_DEBUG constant (true no-op when disabled)
* Note: No behavioral changes — diagnostic-only release for OAuth connection troubleshooting

= 1.2.2 - 2026/03/14 =
* Fix: OAuth token exchange URL corrected to use DEFHO platform API endpoint

= 1.2.1 - 2026/03/14 =
* Fix: OAuth callback URL encoding — removed redundant rawurldecode (WP_REST_Request already decodes)

= 1.2.0 - 2026/03/13 =
* New: One-Click Connect — OAuth 2.0 Authorization Code + PKCE (S256) flow for zero-manual-entry DEFHO connection
* New: DEF_Core_OAuth class — PKCE verifier/challenge generation, transient-based state storage, REST callback endpoint
* New: "Connect to DEFHO" hero button on Connection tab with manual connection as accordion fallback
* Security: OAuth callback session-bound — verifies logged-in admin with def_admin_access matches the initiating user
* Security: PKCE verifier stored server-side in WordPress transient (5-min TTL), consumed immediately on use
* Security: Authorization code exchanged server-side — no secrets exposed to browser
* Improved: Connection tab UI redesign — primary one-click action, manual entry as fallback
* Improved: Success/error admin notices after OAuth redirect
* Fix: Staff AI menu now visible to all DEF roles (management and admin, not just staff)

= 1.1.0 - 2026/03/13 =
* Security: Encrypted storage for connection secrets (API key, service auth secret)
* Security: Authenticated encryption at rest using sodium (XSalsa20-Poly1305) with AES-256-GCM fallback
* Security: Automatic migration of existing plaintext secrets on first read
* Security: Salt rotation detection with admin notice for recovery
* New: Environment-aware API URL resolution (wp-config.php constant, stored option, or platform default)
* New: Complete plugin cleanup on uninstall (all options, transients, and capabilities removed)
* Improved: Admin connection panel shows masked credential status instead of raw values
* Improved: Removed manual API URL input field (managed by platform connection push)
* Improved: Connection config push now stores WordPress site URL correctly
* Refactor: Renamed internal admin API controller class for architectural clarity
* Tests: 35 new encryption tests covering round-trip, auto-migration, malformed data, and salt rotation
