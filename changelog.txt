*** Digital Employee Framework - Core Changelog ***

= 2.0.2 - 2026/04/07 =
* Refactor: Customer Chat — replace anonymous JWT auth with WordPress BFF proxy (Spec V1.3)
* New: /chat/stream proxy endpoint — WordPress proxies chat requests to DEF with trusted headers
* New: SSE streaming proxy via curl CURLOPT_WRITEFUNCTION pass-through
* New: No-silent-downgrade rule — logged-in users with invalid nonce get 403, never anonymous fallback
* Removed: Anonymous token endpoint (/context-token-anonymous) — no longer needed
* Removed: fetchAnonymousToken(), authGeneration, anonSid, generateUUIDv4() from widget JS
* Removed: AnonymousTokenTest.php (188 lines) — replaced by BFF proxy architecture
* Security: Widget no longer manages tokens for chat — identity resolved server-side by WordPress
* Net: +90 / -375 lines — major simplification

= 2.0.1 - 2026/04/06 =
* Fix: Critical — anonymous token obtained then discarded by Promise chain in fetchContextToken()
  - fetchAnonymousToken() returned token string but chained .then() expected response object with .token property
  - Result: anonymous users always got "Unable to connect" despite token being issued correctly
  - Fix: nested res.json() extraction so anonymous fallback path flows directly to caller

= 2.0.0 - 2026/04/06 =
* Fix: Customer Chat — anonymous visitors can now chat without logging in (Bug #1 Issue A)
* Fix: Customer Chat — Send button now enables when typing (Bug #1 Issue B)
* New: Anonymous context token endpoint (GET /context-token-anonymous) — issues channel-scoped JWT for guest users
* New: Anonymous session ID (SID) lifecycle — persisted in localStorage, validated server-side as UUIDv4
* New: Widget anonymous token fallback — authenticated endpoint returns 401, widget seamlessly obtains anonymous token
* New: Auth transition guard (authGeneration counter) — prevents stale anonymous refresh from overwriting authenticated token
* New: Thread claim flow sends anonymous principal ID for server-side ownership verification
* Security: IP-based rate limiting on anonymous token endpoint (30 requests per 5 minutes per REMOTE_ADDR)
* Security: Token fallback restricted to 401 only — 500/network errors do not silently downgrade to anonymous
* Dev: 11 new PHPUnit tests (61 assertions) for anonymous token endpoint
* Spec: Fix Spec V5.0, Build Plan V1.3 — reviewed and approved by ChatGPT + Staff AI
